# ИНСТРУКЦИЯ ПО ЗАПУСКУ

## ❌ НЕПРАВИЛЬНЫЙ СПОСОБ (ОШИБКА)

**Если вы видите ошибку:**
```
the setup command may only be used inside a describe block
```

**Это означает:** Вы пытаетесь запустить PowerShell файл как JavaScript/Node.js файл.

**НЕ делайте так:**
- ❌ `npm run setup.ps1`
- ❌ `vitest run setup.ps1`
- ❌ `node setup.ps1`
- ❌ Открытие файла в редакторе кода и нажатие "Run"

---

## ✅ ПРАВИЛЬНЫЙ СПОСОБ

### Способ 1: BAT файл (рекомендуется)

**Windows 10/11:**
1. Откройте папку `win10_deploy`
2. **Правый клик** на `setup.bat`
3. Выберите **"Запуск от имени администратора"**
4. Дождитесь завершения

### Способ 2: PowerShell вручную

1. **Правый клик** на меню "Пуск" (клавиша Win + X)
2. Выберите **"Windows PowerShell (Admin)"** или **"Терминал (Admin)"**
3. Выполните:

```powershell
cd D:\Opencode\OpenCode_models\Normalize\win10_deploy
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser -Force
.\setup.bat
```

### Способ 3: Поэтапная установка

Если BAT файл не работает, откройте `MANUAL_SETUP.ps1` и выполняйте команды по одной:

```powershell
# Копируйте каждую строку и вставляйте в PowerShell

# 1. Разрешить скрипты
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser -Force

# 2. Установить Chocolatey
Set-ExecutionPolicy Bypass -Scope Process -Force
[System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072
iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

# 3. Установить Node.js
choco install nodejs-lts -y

# И так далее...
```

## 🚀 ПОСЛЕ УСТАНОВКИ

### 1. Настроить API ключи

Отредактируйте `frontend\.env.local`:
```env
VITE_GOOGLE_API_KEY=ВАШ_КЛЮЧ_GOOGLE
VITE_OPENROUTER_API_KEY=ВАШ_КЛЮЧ_OPENROUTER
VITE_API_URL=http://localhost:8000
```

### 2. Запускать серверы

**Через BAT файлы:**
```
Двойной клик на start.bat
```

**Через PowerShell:**
```powershell
.\start.ps1
```

### 3. Открыть в браузере

```
http://localhost:3000
```

## 📞 ПОДДЕРЖКА

Если ничего не работает:
1. Проверьте, что используете **PowerShell**, а не CMD
2. Убедитесь, что запускаете **от имени администратора**
3. Проверьте интернет-соединение
4. Отключите антивирус временно
5. Выполните команды из `MANUAL_SETUP.ps1` вручную
